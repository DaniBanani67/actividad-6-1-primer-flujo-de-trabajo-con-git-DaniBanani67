## Diferencias

El directorio de trabajo es donde modifico los archivos.
La zona de staging es donde selecciono qué cambios quiero guardar.
El historial de commits es donde se almacenan definitivamente los cambios.

## Error típico

Un error común es olvidarse de usar git add antes de hacer commit.
Para evitarlo, siempre reviso git status antes de hacer commit.

## Ventaja del staging

Permite elegir exactamente qué cambios incluir en cada commit, haciendo el historial más claro y organizado.
